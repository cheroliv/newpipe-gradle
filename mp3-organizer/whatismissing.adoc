Ce document liste les fonctionnalités manquantes et améliorations possibles pour le projet MP3 Organizer, organisé par priorité.

== Contexte

Le projet MP3 Organizer est une application **PostgreSQL-centric** qui permet de :

* Scanner des fichiers MP3 depuis de multiples dossiers sources
* Extraire les métadonnées (tags ID3) via JAudiotagger
* Stocker dans PostgreSQL avec fonctions stockées et vues
* Exporter en JSON, YAML, XML, SQL
* Utiliser une IA locale (Ollama + LangChain4j) pour générer du SQL depuis un prompt

== 2. Important (fonctionnalités manquantes)

=== 2.1. Scan incrémental

[cols="1,2,1",options="header"]
|===
|Manque |Pourquoi |Priorité
|===

* Rescan de TOUS les fichiers à chaque exécution
* Lent pour 2308+ fichiers

[source,sql]
----
-- Solution: comparer avec l'historique
SELECT file_path, file_size_bytes, updated_at
FROM tracks
WHERE updated_at < NOW() - INTERVAL '1 day'
----

=== 2.2. Détection de doublons

[cols="1,2,1",options="header"]
|===
|Manque |Pourquoi |Priorité
|===

* Pas de détection fichiers dupliqués
* Même chanson dans plusieurs dossiers
* Gaspillage d'espace disque

[source,kotlin]
----
// Hash MD5/SHA256 pour détection
fun File.md5(): String {
    val md = MessageDigest.getInstance("MD5")
    return inputStream().use { 
        it.copyTo(DigestOutputStream(md)) 
    }
    md.digest().toHexString()
}
----

=== 3.2. Dockerfile

* Containeriser l'application Kotlin
* Multi-stage build pour image légère
* Docker Compose avec PostgreSQL

[source,dockerfile]
----
# Dockerfile
FROM gradle:8.6-jdk17 AS build
COPY . /app
WORKDIR /app
RUN gradle build --no-daemon

FROM eclipse-temurin:17-jre-alpine
COPY --from=build /app/build/libs/*.jar /app/mp3-organizer.jar
ENTRYPOINT ["java", "-jar", "/app/mp3-organizer.jar"]
----

}

== 7. Conclusion

Le projet est **fonctionnel** mais nécessite des améliorations pour une utilisation en production :

1. **Priorité absolue** : Tests + Configuration externalisée
2. **Ensuite** : Scan incrémental + Batch inserts (performance)
3. **Optionnel** : API REST + Web UI (expérience utilisateur)

[quote]
Le code est un jardin : il faut le cultiver régulièrement, pas juste le planter une fois.
